## ADDED Requirements

### Requirement: Runtime exports a structured coverage data package

The Angelscript runtime SHALL export code coverage as a machine-readable data package instead of relying on runtime-generated HTML as the primary artifact.

#### Scenario: Coverage data package is written

- **WHEN** code coverage recording is stopped with an output directory
- **THEN** the runtime SHALL write a parseable `coverage_summary.json` data package in that directory
- **AND** the package SHALL include `schema_version`, generator metadata, generation time, global summary, file entries, directory aggregation, settings, capture capabilities, and extension fields

#### Scenario: Data package supports current manual report information

- **WHEN** a consumer reads the coverage data package
- **THEN** the consumer SHALL be able to determine total coverage, per-file coverage, per-line hit counts, source line counts, directory grouping, and summary inclusion status without reading runtime-generated HTML

#### Scenario: Runtime HTML output is no longer required

- **WHEN** code coverage data is exported after this change
- **THEN** correctness SHALL be defined by the structured data package rather than by `index.html` or `*.as.html` files generated by runtime C++

### Requirement: Coverage export is deterministic and tool-friendly

The coverage data package SHALL use stable field names and deterministic ordering so CI, tests, and external tools can consume it reliably.

#### Scenario: File and line ordering is stable

- **WHEN** equivalent coverage data is exported across repeated runs
- **THEN** file entries SHALL be ordered deterministically by path
- **AND** line entries within each file SHALL be ordered by line number

#### Scenario: Zero executable lines are explicit

- **WHEN** a coverage summary or file has no executable lines
- **THEN** the data package SHALL distinguish that state from real zero-percent coverage with an explicit `has_executable_lines` or equivalent field

#### Scenario: Exclude patterns are observable

- **WHEN** files are excluded from coverage summary calculations
- **THEN** the data package SHALL include the active exclude patterns and per-file inclusion or exclusion status

## Testing Requirements

- Target test layer: Runtime Integration.
- Expected Automation prefix: `Angelscript.TestModule.Core.CodeCoverage.*`.
- Recommended helper/harness: existing `FAngelscriptCodeCoverage` tests in `Plugins/Angelscript/Source/AngelscriptTest/Core/` with JSON parsing assertions.
- Verification entry point: `powershell.exe -NoProfile -ExecutionPolicy Bypass -File Tools\RunTests.ps1 -TestPrefix "Angelscript.TestModule.Core.CodeCoverage" -Label code-coverage-data-export -TimeoutMs 600000`.
