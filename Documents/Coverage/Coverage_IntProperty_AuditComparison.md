#### 2. 继承和 default 关键字
**审计报告：** ⬜ P2 中优先级
**当前状态：** ⬜ 仍未实现
**示例：**
```angelscript
class Base {
    UPROPERTY()
    int BaseValue = 10;
}

class Derived : Base {
    UPROPERTY()
    int DerivedValue = 20;
    
    default BaseValue = 100;  // override parent default
}
```

**评估：**
- ✅ 有独立测试文件（AngelscriptSyntaxDefaultStatementTests.cpp）
- ✅ 不是 int 特有的，是语法特性
- ✅ 可选补充项，非核心

**建议：** 保持现状，作为独立的语法测试

#### 3. 数字分隔符语法
**审计报告：** ⬜ P3 低优先级

**示例：**
```angelscript

```

**评估：**
- ⬜ 需要先确认 AngelScript 是否支持此语法
- ✅ 如果支持，是字面量的一部分
- ⬜ 不影响核心覆盖率

**建议：** 可选，需要语法确认

#### 4. USTRUCT 嵌套路径
**审计报告：** ⬜ P3，需要独立 USTRUCT 框架
**当前状态：** ⬜ 未实现
**示例：**
```angelscript

struct FMyStruct {
    UPROPERTY()
    int NestedInt;
}

UCLASS()
class AMyActor : AActor {
    UPROPERTY()
    FMyStruct MyStruct;  // 访问: MyStruct.NestedInt
}
```

**评估：**
- ✅ 不是 int 特有的问题
- ✅ 需要先建立 USTRUCT 覆盖框架
- ✅ 独立规划项

**建议：** 保持现状，作为 USTRUCT 覆盖的一部分
